# 首页引导

## 关于 PassNAT

:::tip
PassNAT 使用 [fatedier/frp](https://github.com/fatedier/frp) 进行开发，如果此开源项目对你有帮助，可以点击链接给作者点星支持。
:::

PassNAT 是一个以 Go-Frp 为核心，使用**原生接口**构建的 frp 托管平台。

因为是原生接口构建，不修改客户端与服务端，兼容性较好。如有已用的 frp 可以无缝迁移，只需更换配置文件即可。

### 为什么选 PassNAT

- PassNAT 致力于打造用户优先的交互体验，在 UI 上秉承效率第一的原则设计风格打造现代化的控制面板，兼容移动端适配。
- 优质线路来自于 [Opticalvm](https://www.opticalvm.com) 云计算团队多年运营经验特别甄选，以 `高防` `大带宽` `多线BGP` `高稳定` 为标准制定节点标签。
- Frp 稳定的核心最重要的在于节点的质量与带宽，[Opticalvm](https://www.opticalvm.com) 团队的加入直接保障了总节点的高稳定与高带宽。

### 价格为什么这么有优势

- 我们本身就是云计算团队，经营着自己的 IDC 业务 [原光云计算](https://www.opticalvm.com)。不管是节点的质量还是带宽的成本都是我们自己承担，所以我们可以提供尽可能优惠的价格。
- 其次 FRP 是一款开源软件，PassNAT 只是按照官方接口进行开发。极大减少了开发成本，所以我们可以提供更优惠的价格。
- 个人版套餐的价格是我们的成本价，我们希望能够帮助更多的用户，所以我们提供了这个价格。但是这个价格是不包括任何技术支持的，如果你有技术问题，可以在社区提问，我们会尽量帮助你。

我们希望通过 PassNAT 的努力，让更多的用户能够体验到高质量的 FRP 服务。

## 文档专业名词解释

### 节点

这是指运行 frp 服务器（frps）的主机。它通常是位于公网上的服务器，负责接受来自客户端的流量并将其转发到相应的公网服务。
节点的网络质量直接影响到 frp 的稳定性和性能。 PassNAT 会提供多个节点供用户选择。各套餐均配备 `BGP` 节点，保证跨网链路稳定性。

### 隧道

这是指客户端与节点之间的连接。隧道是 frp 的核心概念，隧道包含了客户端与节点之间的通信协议、加密方式、传输协议等信息。
